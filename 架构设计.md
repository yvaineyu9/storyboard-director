## 0. 约束理解、风险与开放项（先对齐，再落架构）

### 0.1 我对硬约束的理解（逐条复述，确认无偏差）

| # | PRD 硬约束 | 架构落点 |
|---|---|---|
| 1 | 意图在前（顺序、含追问），context 足够后才并行 | A1 单会话顺序执行 + 追问循环；A1 完成且 `ready=true` 后才 `Promise.all` 启 A2/A3 |
| 2 | 死模板＝静态资产，生产只调用、不生成不改写；A0 一次性引导后退场 | A0 离线脚本产出 `assets/storyboard-library.json` 并落盘；运行时只读不写；A2 只输出模板编号 |
| 3 | 模板 4 字段：①图片构图（二维图形本身）②分镜说明 ③决策指引 ④场景适配 | 死模板 schema 见 §3；①落为**归一化绘制原语数组** `figure`，渲染器只解释、不重算 |
| 4 | 两层色仅 `#000`+`#40FF5E`，Canvas 写死 hex 不用 token | 渲染器内 `BG='#000'` / `FG='#40FF5E'` 两个常量；UI 卡片用 token，Canvas 不用 |
| 5 | 分镜规模硬约束 3~6 镜 | A1 产出 `shotCount∈[3,6]` 并下发给 A2/A3；A4 再做 `clamp(3,6)` 兜底 |
| 6 | 跨模板混搭护栏：运镜手法 ≤2~3 种、情绪基调单一 | A2 system prompt 注入护栏；A4 校验 `distinct(camera)≤3 && 单 mood`，越界回退 |
| 7 | 全走 AIUI 官方 API + 设计规范 | 语音 `SpeechRecognition`；A1–A4 端侧 `LanguageModel`（A2/A3 双独立会话）；`<streamdown>` + `<canvas>`/`wx.createCanvasContext`；480/黑底/卡片/12px 圆角/2px 边框/无 emoji |

### 0.2 关键发现：`~/AIUI/samples/voice-agent` 是本项目的「半成品雏形」

样例已具备、**可直接复用**：
- `pages/index/index.ink`：`SpeechRecognition` 全生命周期处理（interim/final/error/end）、`<streamdown>` 流式渲染、idle/listening/thinking/responding/error 状态机骨架、480px 黑底卡片样式、硬件键 `onKeyDown` 切麦。
- `lib/storyboard-templates.js`：**受控词表**（`SHOT_SIZES`/`CAMERA_MOVES`/`COMPOSITIONS`/`TRANSITIONS`/`MOODS`）、`resolveShot`/`resolveBoard`/`buildMarkdown`/`buildEditingFlow` 等 id→标签→Markdown 的确定性工具。

样例与 PRD 的**关键偏差**（本架构要补齐的核心增量）：
1. 样例是**单个 LLM 会话**一把出整板；PRD 要求 **A1 顺序前置（含追问）→ A2/A3 双会话并行 → A4 汇合**。
2. 样例的 `ARCHETYPES` 是"可整体套用或按场景改编"的**风格原型**（对应 PRD §5 的 T1–T8 风格参考），**不是** PRD §4 的「死模板（4 字段 + 可绘制构图）」。需新增 A0 产出的死模板库。
3. 样例**没有 Canvas 渲染**、没有两层色胶片条。
4. 样例**没有 A1 追问环节**。

> 结论：把 `voice-agent` 作为脚手架起点 fork，保留语音/streamdown/状态机/词表与 resolve 工具，新增 A0 死模板库、A1 追问、A2/A3 并行编排、A4 确定性组合、Canvas 胶片条渲染。

### 0.3 风险与歧义（标记为开放项，建议但不擅改锁定决策）

| 编号 | 风险 / 歧义 | 影响 | 架构建议（默认采用，待 PM 确认） |
|---|---|---|---|
| OQ-1 | A2 与 A3 并行时**互不知镜数**，两路长度可能不一致 | 组合错位 | **A1 产出 `shotCount` 作为唯一镜数真相**，下发两路；A4 按 index 对齐 + 截断/补齐兜底。**（已纳入本架构）** |
| OQ-2 | PRD §9 要求 A3 走 LLM；但模块B 是确定性映射表，LLM 反而引入 JSON 解析风险 | 稳健性 | 保持 A3 为 LLM 会话（遵 PRD），但**内置模块B 确定性回退表**：A3 解析失败即用纯 JS 映射兜底。等于把降级路径做进主流程。 |
| OQ-3 | PRD §8.1 把 A4 列为 Agent；但"对齐+约束+生成示意图数据+指引"全是确定性逻辑 | 稳健性 | **【PM 已确认 2026-06-06】A4 实现为纯 JS（无 LLM 调用）**，与 PRD §8.2「结果文字由 A4 数据确定性生成 Markdown」一致。 |
| OQ-4 | 端侧 `LanguageModel` **结构化 tool-call 未在 JS 层暴露**（apis-ai.md 明确） | 数据契约 | 所有 Agent 走"**system prompt 约束词表 + 纯文本返回 JSON + 稳健解析**"，不依赖 tool-call。见 §6.1。 |
| OQ-5 | `LanguageModel.availability()` 不可用 / `wx.createCanvasContext` 返回 null | 能力降级 | 分级降级策略，见 §6.2。 |
| OQ-6 | 死模板数量＝10 构图 × 景别 × 镜头调度 × 情绪 叉乘，可能爆量、且 system prompt 装不下全量索引 | A2 选型成本 | **【PM 已确认 2026-06-06】A0 收敛到精选集 18~24 条，每条必含情绪标签（`sceneFit.moods`）+ 场景（`sceneFit.scenes`）**；A2 注入的是**精简索引**（id+分镜说明+决策指引+场景适配，不含 figure 原语）。见 §3.3。 |
| OQ-7 | PRD §2⑤「分镜指引（逐镜可执行拍摄指引）」与 §7(d)「简要说明」是否同一物？ | 输出结构 | **【PM 已确认 2026-06-06】分镜指引不做 LLM 润色**——它最终用于「确定具体采用哪个分镜」，是**决策/可执行导向**而非散文。§7(d) 简要说明＝A2 每镜 `description`；§2⑤ 分镜指引＝A4 用受控词表 `desc` **确定性拼装**的逐镜可执行清单。二者并存、来源不同、均无 LLM。 |
| OQ-8 | 追问轮次上限 PRD 写「1~2 轮」 | 体验 | 取 `MAX_CLARIFY=2`；缺关键项（主体/场景/情绪三者）才问，详见 §6.3。 |
| OQ-9 | 单个 `<canvas>` 画整条胶片条 vs 每镜一个 canvas | 渲染/性能 | 采用**单 canvas 画整条 + 横向 `scroll-view scroll-x`**（节点少、绘制一次、对齐稳）。见 §4。 |

```
storyboard-director/                  # fork 自 samples/voice-agent
├── AGENTS.md                         # agent 清单：权限 microphone/network，技能 storyboard-director
├── app.json                          # pages 路由 + window；viewport width=device-width
├── app.js                            # 生命周期（沿用样例）
├── package.json
│
├── assets/
│   └── storyboard-library.json       # 【A0 落盘产物 · 死模板库】生产只读，运行时不写

├── lib/
│   ├── vocab.js                      # 受控词表（从样例 storyboard-templates.js 抽出 SHOT_SIZES/CAMERA_MOVES/COMPOSITIONS/TRANSITIONS/MOODS）

│   ├── figure.js                     # 构图原语生成器（仅 A0 离线用）+ 原语类型定义；运行时不依赖
│   ├── agents/
│   │   ├── intent.js                 # A1：意图分析 + 追问循环（单会话）
│   │   ├── composition.js            # A2：画面→构图（管线 a，独立会话，调用死模板）
│   │   ├── rhythm.js                 # A3：情绪→剪辑节奏（管线 b，独立会话，模块B 映射 + 确定性回退）
│   │   └── combine.js                # A4：纯 JS 组合（对齐/护栏/3~6/示意图数据/分镜指引/Markdown）
│   ├── json-rescue.js                # 稳健 JSON 解析（剥 ```、提取首个 {}、容错）
│   └── renderer/
│       └── filmstrip.js              # 两层色胶片条 Canvas 渲染器（解释 figure 原语 + 节奏层 + 转场符号）
│
├── pages/
│   └── index/
│       └── index.ink                 # 单页 SFC：UI 状态机 + 编排 A1→[A2||A3]→A4 + streamdown + canvas
│
└── tools/                            # 【离线，不打包进生产运行时】
    └── bootstrap/
        ├── a0-generate.md            # A0 生成 prompt（喂模块A + 词表 + 8 原型）
        ├── a0-build.js              # A0 离线脚本：调 LLM 产模板 → figure.js 烘焙原语 → 写 assets/storyboard-library.json
        └── a0-validate.js           # 校验落盘库（编号唯一、4 字段齐、figure 合法、词表合法）
```

设计要点：
- **离线/运行时硬隔离**：`tools/bootstrap/` 与 `lib/figure.js` 只在 A0 引导阶段使用；生产页面只 `import` `lib/library.js`（读 JSON）+ `lib/renderer/filmstrip.js`（解释原语）。生产代码里**不存在任何"生成模板"的调用路径**。
- `lib/vocab.js` 是 A0、A1、A2、A3、A4、渲染器、Markdown 的**单一词表真相**，所有 LLM 只能吐词表里的 id。
- 单页应用（PRD §8.2），状态机全在 `pages/index/index.ink` 的 `<script setup>`。

## 2. Agent 层数据契约与会话编排

### 2.1 编排时序（顺序/并行边界，严格按 PRD §2）

```
语音 final text
      │
      ▼  ── 会话1（顺序，可多轮）──
   A1 意图分析  ──ready?── no ──▶ 追问(streamdown) ──▶ 重新聆听 ──┐
      │ yes                                                     │
      │  intent{ shotCount, mood, ... }            (≤2 轮，回灌 A1)┘
      │
      ├──────────────── 并行屏障：Promise.all ────────────────┐
      ▼ 会话2（独立）                                  ▼ 会话3（独立）
   A2 画面→构图                                    A3 情绪→剪辑节奏
   输入: intent                                    输入: {mood, shotCount}
   输出: shotsA[ {templateId, beat, description} ] 输出: shotsB[ {duration,cutSpeed,transition,pause} ]
      └──────────────── 汇合（await 两路）─────────────────┘
      ▼  ── 纯 JS，无 LLM ──
   A4 组合：按 index zip → clamp(3,6) → 护栏校验 → resolve(id→figure/labels)
      → board → buildMarkdown / buildFilmstripModel / buildGuidance
      ▼
   UI：<streamdown>(卡片) + <canvas>(胶片条) + 指引列表
```

会话管理规则（依 apis-ai.md）：
- A1：`LanguageModel.create({ initialPrompts:[{role:'system',content:INTENT_SYS}] })` 一个会话；追问用同一会话 `prompt()` 多轮（保留上下文）；A1 收尾后 `destroy()`。
- A2、A3：**两次独立 `LanguageModel.create()`**，各自 `prompt()` 一次，`Promise.all([a2.run, a3.run])` 并发；各自 `finally` 内 `destroy()`。
- `system` 只能作 `initialPrompts` 首条；逐轮 `prompt()` 只能 `user`/`assistant`（apis-ai.md 已确认），故词表/护栏全部注入到 `initialPrompts` 的 system。
- 任一会话 `prompt()` 进行中不可在同会话再发——A2/A3 用**不同会话实例**规避。

### 2.2 A1 意图分析 · 输入/输出契约

输入：ASR final text（string）。

输出 JSON（稳健解析）：

```jsonc
{
  "ready": true,                 // false=信息不足需追问
  "ask": "",                     // ready=false 时给用户的追问话术（中文，无 emoji，≤1 句）
  "missing": [],                 // ["subject"|"scene"|"mood"] 缺失项，仅诊断用
  "intent": {
    "subject": "独自旅行的女孩",   // 主体（必需）
    "scene": "黄昏的公路",         // 场景（必需）
    "visualIntent": "辽阔、出发感", // 画面意图（自由文本，喂 A2）
    "mood": "epic",              // 情绪基调（必需，∈ MOODS id）
    "shotCount": 5               // 镜数真相，∈[3,6]；缺省按情绪给默认值
  }
}
```

约束：`mood` 必须是 `vocab.MOODS` 的 id；`shotCount` 整数 3~6；`subject/scene/mood` 三者任一缺失 → `ready=false`（见 §6.3 追问规则）。

### 2.3 A2 画面→构图（管线 a）· 输入/输出契约

输入：A1 的 `intent`（含 `shotCount`）。
注入 system：受控词表 + **死模板精简索引**（`buildLibraryIndexForPrompt()`：每条 = `id｜分镜说明｜决策指引｜场景适配(moods,scenes)`，**不含 figure 原语**）+ 跨模板混搭护栏。

处理：LLM 分析画面意图 → 组关键词 → 按「决策指引/场景适配」**为每镜选 1 个死模板编号**（跨模板混搭），不现编、不改写构图。

输出 JSON：

```jsonc
{
  "shots": [
    {
      "templateId": "T07",          // 必须是库中存在的编号；A4 校验，非法→最近邻回退
      "beat": "establish",          // 叙事节拍提示(可选): establish|build|climax|breath|close
      "description": "黄昏公路向远方延伸，镜头缓慢前推，远端小小身影"  // §7(d) 简要说明，须含镜头调度语言
    }
    // … 长度 = intent.shotCount
  ],
  "cameraUsed": ["fixed","dolly","track"]  // 自报运镜手法集合，供 A4 护栏校验(≤3)
}
```

> 护栏由 system prompt 约束 + A4 兜底双保险：`distinct(每镜模板的 camera) ≤ 3`、`mood` 单一（沿用 A1 的 mood，不在 A2 改）。

### 2.4 A3 情绪→剪辑节奏（管线 b）· 输入/输出契约

输入：`{ mood, shotCount }`（来自 A1）。
注入 system：受控词表 + **模块B 情绪→节奏映射表**（duration 区间 / cutSpeed 档 / transition 偏好 / pause 策略）+ "镜数收敛到 shotCount、给出节奏弧线（建立长→高潮短→收尾留白）"。

输出 JSON：

```jsonc
{
  "shots": [
    {
      "duration": 5,            // 秒，数字
      "cutSpeed": "slow",       // slow|medium|fast|veryfast
      "transition": "dissolve", // ∈ TRANSITIONS id，指到下一镜的衔接；末镜=收尾方式
      "pause": false            // 是否本镜前留白蓄势（高潮前=true）
    }
    // … 长度 = shotCount
  ]
}
```

确定性回退（OQ-2）：A3 解析失败或长度不符 → `rhythm.js` 内置 `MODULE_B[mood]` 纯 JS 映射，按 shotCount 生成节奏弧线，保证主流程不中断。

### 2.5 A4 组合（纯 JS，无 LLM）· 处理与输出

步骤：
1. **对齐**：`zip(shotsA, shotsB)` 按 index；以 `intent.shotCount` 为准，长则截断、短则用回退补齐（A2 用最近邻模板、A3 用模块B 回退）。
2. **约束**：`N = clamp(shots.length, 3, 6)`。
3. **护栏校验**：`distinct(camera) > 3` → 把超出的镜替换为已用 camera 的最近邻模板；强制单 mood。
4. **解析**：每镜 `templateId → library.getById()` 取 `figure`/`composition`/`shotSize`/`camera` 标签；合并 A3 的 `duration/cutSpeed/transition/pause`。
5. **产物**：
   - `board`（§7 最终输出结构，见下）
   - `markdown`：`buildMarkdown(board)`（确定性，供 `<streamdown>`）
   - `filmstripModel`：`buildFilmstripModel(board)`（供 Canvas 渲染器，见 §4.2）
   - `guidance[]`：`buildGuidance(board)`（§2⑤ 逐镜执行指引，用 vocab.desc 拼装；**决策/可执行导向，无 LLM 润色**——目的是让用户据此确定具体采用哪个分镜，OQ-7 PM 已确认）

最终 `board` 数据结构（对外唯一，对齐 PRD §7）：

```jsonc
{
  "title": "黄昏公路·出发",
  "mood":  { "id": "epic", "label": "史诗壮阔" },
  "templates": ["T07","T02","T03","T07","T11"],   // 所用模板编号混搭列表
  "shotCount": 5,                                   // (a) 卡片顶部明示，3~6
  "totalDuration": 19,
  "shots": [
    {
      "index": 1,
      "title": "镜头 1",
      "duration": 5,                                // (b) 每镜时长
      "shotSize":    { "id":"wide",  "label":"远景" },
      "composition": { "id":"leading","label":"引导线" },
      "camera":      { "id":"dolly", "label":"移镜头" },
      "transition":  { "id":"dissolve","label":"叠化" },
      "cutSpeed": "slow",
      "pause": false,
      "templateId": "T07",
      "figure": { "primitives": [ /* 见 §3.2 */ ] },// 用于 Canvas
      "description": "黄昏公路…镜头缓慢前推…"          // (d) 简要说明
    }
    // …
  ],
  "editingFlow": [                                   // (c) 串联剪辑，独立数组
    { "step":1, "shot":"镜头 1", "duration":5, "transition":"dissolve", "transitionLabel":"叠化", "next":"镜头 2" }
    // …
  ],
  "guidance": [                                      // §2⑤ 逐镜可执行指引
    "镜1 · 远景/引导线/移镜头：机位低、用公路两侧汇聚线引导视线，缓慢前推 5s，叠化入下一镜"
    // …
  ]
}
```

## 3. 死模板库落盘格式（A0 产物 · 静态资产）

### 3.1 文件：`assets/storyboard-library.json`

```jsonc
{
  "schemaVersion": 1,
  "generatedBy": "A0-bootstrap",         // 标记来源；生产只读
  "palette": { "bg": "#000", "fg": "#40FF5E" },
  "templates": [ /* 见 3.2，建议 18~24 条 */ ]
}
```

### 3.2 单条死模板 schema（每条严格 4 字段，编号化）

```jsonc
{
  "id": "T07",                                 // 编号，全库唯一

  // ① 图片构图：二维图形「本身」——归一化绘制原语（0..1 坐标），渲染器只解释、不重算
  "figure": {
    "viewBox": [1, 1],                         // 归一化坐标系
    "primitives": [
      { "kind": "line",     "from": [0.10,0.62], "to": [0.50,0.40], "width": 0.012 },
      { "kind": "line",     "from": [0.90,0.62], "to": [0.50,0.40], "width": 0.012 },
      { "kind": "circle",   "at": [0.50,0.40],   "r": 0.05,  "fill": true },
      { "kind": "rect",     "at": [0.00,0.74],   "size": [1.0,0.26], "fill": false }
      // kind ∈ line | rect | circle | polyline ；fill 默认 false（描边）
      // 颜色不在原语里：bg 永远 #000、所有原语永远 #40FF5E（两层色硬约束）
    ]
  },

  // ② 分镜说明：构图法则 + 景别 + 镜头调度（全部用 vocab id，标签由渲染层解析）
  "caption": { "composition": "leading", "shotSize": "wide", "camera": "dolly" },

  // ③ 决策指引：给 A2 选型参考（什么时候调用、解决什么画面问题）
  "decisionHint": "表达出发、纵深、代入感；公路/通道/走廊类画面首选，用汇聚线把视线引向远端主体",

  // ④ 场景适配：适配情绪 + 场景
  "sceneFit": { "moods": ["epic","calm"], "scenes": ["公路","走廊","行进中"] }
}
```

字段与 PRD §4.1 的逐项对应：①图片构图＝`figure`（原语＝可直接绘制的二维图形）②分镜说明＝`caption`（构图法则/景别/镜头调度）③决策指引＝`decisionHint` ④场景适配＝`sceneFit`。

### 3.3 A0 生成与覆盖策略

- 输入：模块A 的 10 种构图 + PRD §10 受控词表（景别/镜头调度/情绪场景）+ §5 的 T1–T8 风格原型（仅风格参考，不进生产）。
- 叉乘后**收敛为精选集 18~24 条**（OQ-6，PM 已确认）：覆盖 10 构图 × 关键景别/运镜组合 × 8 情绪标签。**每条模板都是硬性必含 `sceneFit.moods`（情绪标签，≥1，∈ MOODS）+ `sceneFit.scenes`（场景，≥1）**——这是 A2 按「场景适配」选型的依据，缺一即 A0 校验不通过。确保每个 `mood` 至少有 3~4 条可选模板（保证 A2 跨模板混搭有料）。
- `figure.primitives` 由 `lib/figure.js` 的构图生成器**烘焙成字面量原语**写入 JSON（A0 离线跑一次）；运行时渲染器不再依赖生成器。
- 落盘后 `a0-validate.js` 校验：编号唯一、4 字段齐全、`caption`/`sceneFit` 取值全在 vocab 内、`figure.primitives` 坐标 ∈[0,1] 且 kind 合法。
- 扩充＝重跑 A0（离线维护），不在生产流程内。

## 4. 两层色「胶片条」Canvas 渲染方案

### 4.1 节点与坐标常量（仅两色，hex 写死）

```
组件：<scroll-view scroll-x><canvas id="filmstrip" width="{{stripW}}" height="{{stripH}}"/></scroll-view>
绘制：lib/renderer/filmstrip.js → drawFilmstrip(model)，内部 wx.createCanvasContext('filmstrip')
```

常量（px，backing 像素）：

| 常量 | 值 | 含义 |
|---|---|---|
| `PAD` | 12 | 画布四周留白 |
| `CELL` | 120 | 单镜构图格边长（正方画框） |
| `GUT` | 30 | 镜间间隙（画转场符号） |
| `RHY_H` | 26 | 节奏层条带高度 |
| `RHY_GAP` | 10 | 构图格与节奏条间距 |
| `LW` | 2 | 主描边线宽（对齐设计规范 2px） |
| `BG` | `'#000'` | 底层（唯一背景色） |
| `FG` | `'#40FF5E'` | 形层（唯一主体色） |

画布尺寸（N 镜）：
```
stripW = PAD*2 + N*CELL + (N-1)*GUT
stripH = PAD*2 + CELL + RHY_GAP + RHY_H
```
（N=6 时 stripW = 24 + 720 + 150 = 894，靠 scroll-x 横滑；stripH ≈ 24+120+10+26 = 180）

### 4.2 `filmstripModel`（A4 → 渲染器的输入）

```jsonc
{
  "cells": [
    {
      "index": 1,
      "figure": { "viewBox":[1,1], "primitives":[…] },   // 来自死模板，原语
      "duration": 5,
      "cutSpeed": "slow",                                 // 决定节奏条 tick 密度
      "transitionToNext": "dissolve",                     // 末镜为 null
      "pause": false
    }
  ],
  "maxDuration": 5                                         // 归一化节奏条长度用
}
```

### 4.3 绘制规范（逐层，两色）

绘制顺序（整画一次，幂等）：

1. **底**：`fillStyle=BG; fillRect(0,0,stripW,stripH)`。
2. **逐格**（cell i，原点 `ox = PAD + i*(CELL+GUT)`，`oy = PAD`）：
   - **画框**：`strokeStyle=FG; lineWidth=LW; strokeRect(ox,oy,CELL,CELL)`。
   - **序号**：`fillStyle=FG; font='12px Arial'; textBaseline='top'; fillText(String(index), ox+4, oy+4)`。
   - **构图原语**：`save(); translate(ox,oy)`，把每个归一化原语 `× CELL` 映射进格内，`strokeStyle=FG/fillStyle=FG` 画线/矩形/圆/折线（`fill` 决定描边或填充，线宽 `width*CELL` 或 `LW`）；`restore()`。
   - **节奏层**（格下方 `ry = oy+CELL+RHY_GAP`）：
     - 条长 `barLen = CELL * clamp(duration/maxDuration, 0.18, 1)`。
     - 底线：`FG` 描边一条水平基线 `(ox,ry+RHY_H/2)→(ox+barLen,ry+RHY_H/2)`。
     - **密度＝切换速度**：按 `cutSpeed` 取 tick 间距 `{slow:18, medium:12, fast:8, veryfast:5}`，在 barLen 上等距画竖向短刻度（`FG`）——越密＝切得越快。
     - **留白蓄势**：`pause===true` 时在条首画一段虚线/空缺（`setLineDash([3,4])` 画前导静音段）。
   - **转场符号**（非末镜，画在右侧 GUT 中心 `gx = ox+CELL+GUT/2`，`gy = oy+CELL/2`）：
     | transition | 画法（FG 描边） |
     |---|---|
     | `cut` 硬切 | 一条竖线 │ |
     | `dissolve` 叠化 | 交叉 ✕（两条对角线） |
     | `fade` 淡入淡出 | 空心圆 ○ |
     | `match` 匹配剪辑 | 两个同向 chevron 》》 |
     | `whip` 甩切 | 斜纹束 ⫽（3 条平行斜线） |
     | `jcut` J切 | 竖线 + 下方短横（声音先入） |
     | `lcut` L切 | 竖线 + 上方短横（声音延留） |
3. **flush**：绘制结束 `ctx.flush()`。

> 全程只用 `BG`/`FG` 两个 hex；无渐变、无第三色、无 emoji。apis-canvas.md 已确认 `#rrggbb`/`fillRect`/`strokeRect`/`arc`/`moveTo`/`lineTo`/`setLineDash`/`fillText`/`flush` 均支持。

### 4.4 渲染时机与容错

- canvas 节点需先挂载：切到 `result` 态 `setData` 后，下一渲染帧节点才存在。
- 模式：`drawFilmstrip` 内 `const ctx = wx.createCanvasContext('filmstrip'); if(!ctx) retry`——最多重试 5 次（每次 ~16ms）等节点就绪（apis.md：节点不存在时返回 null）。
- 每次出新结果**全量重画**（先 `clearRect`/`fillRect` 铺底），不做增量。

## 5. UI 状态机与组件映射

### 5.1 状态机（扩展自 voice-agent 样例，对齐 PRD §8.2）

```
        ┌──────────────────────────────────────────────┐
        ▼                                                │
     idle ──tap/key──▶ listening ──final──▶ analyzing.intent
        ▲                  │ err                  │
        │                  ▼                       ├─ready=false─▶ clarifying ──tap──▶ listening

        │                  │ retry                  └─ready=true──▶ analyzing.parallel

                                                                        ▼
                                                              analyzing.combine (A4, JS)
                                                                        ▼
                                                                     result ──tap(新一条)──▶ listening
```

`phase` 取值：`idle | listening | clarifying | analyzing | result | error`；`analyzing` 再带 `stage ∈ {intent, parallel, combine}` 驱动进度点文案。

### 5.2 状态 → 组件映射

| result 结果 | Markdown 卡片(标题/情绪/模板编号/a–d) + 逐镜胶片条 + 指引列表 | `<streamdown content="{{markdown}}">` + `<canvas id="filmstrip">` in `<scroll-view scroll-x>` + 指引 `<view ink:for>` | 部分复用 |

## 6. 稳健性设计

### 6.1 LLM 输出 JSON 解析容错（`lib/json-rescue.js`）

端侧 `LanguageModel` 无结构化 tool-call（OQ-4），全部走文本→JSON。解析管线：

1. 去除 ```json / ``` 代码围栏与首尾噪声。
2. 截取**第一个平衡的 `{ … }`** 子串（栈匹配花括号）。
3. `JSON.parse`；失败则尝试常见修复（去尾逗号、单引号→双引号）。
4. 仍失败 → 返回 `null`，交由各 Agent 的**领域级回退**：
   - A1 失败 → 视为 `ready=false`，用通用追问话术「能再说说你想拍的主体、场景和情绪吗？」
   - A2 失败 → 用 `intent.mood` 选 1 个最匹配的风格原型 archetype 直接铺 `shotCount` 镜（最近邻模板）。
   - A3 失败 → 模块B 确定性映射表（OQ-2）。
5. 每个 Agent 都做**字段级清洗**：id 不在词表 → 取 vocab 默认值（沿用样例 `pick(map,id,fallback)` 思路）；`shotCount`/`duration` 数值越界 → clamp。

### 6.2 能力降级（OQ-5）

| 能力 | 检测 | 降级 |
|---|---|---|
| `SpeechRecognition` 缺失 | `typeof SpeechRecognition === 'undefined'`（样例已做） | 进 `error`，提示运行时不支持语音；（可选）开放文本输入兜底 |
| `LanguageModel.availability() !== 'available'` | A1 启动前检测 | 进 `error`「AI 运行时暂不可用」；不进入并行 |
| A2/A3 单路失败（会话报错） | `try/catch` + `Promise.allSettled` | 失败路走领域回退（A2→archetype 最近邻 / A3→模块B 表），另一路成功则照常组合 |
| `wx.createCanvasContext` 返回 null | 重试 5 次仍 null | 胶片条降级为**纯文字版**（Markdown 内已含构图/景别/运镜/时长/转场），不阻断结果展示 |
| 死模板库加载失败 | `loadLibrary()` catch | 进 `error`「模板库缺失，请重跑引导」（生产期理应已落盘） |

并发用 `Promise.allSettled([a2,a3])` 而非 `all`，保证单路失败不连坐。

### 6.3 追问触发规则（OQ-8，细化 PRD §3「智能判断」）

- 关键三项：`subject`（主体）/ `scene`（场景）/ `mood`（情绪）。
- 触发：A1 判定**任一关键项缺失或置信不足** → `ready=false`，针对缺失项发**一句**定向追问（只问缺的，不泛问）。
- 上限：`MAX_CLARIFY = 2`。计数器在页面态维护；达上限仍不全 → 用已有信息 + 默认值（`mood` 缺→按 visualIntent 猜，再不行取 `calm`；`shotCount` 缺→按 mood 默认）强制 `ready=true` 进入并行，不再纠缠。
- 追问话术由 A1 产出 `ask` 字段（中文、≤1 句、无 emoji），UI 用 `<streamdown>` 展示后回到 `listening`，新转写**追加**回灌同一 A1 会话。
