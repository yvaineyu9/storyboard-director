# 分镜导演 Agent · 产品需求文档（PRD）

- 版本：v1.2（需求定稿；订正：分镜模板为死模板/静态资产）
- 日期：2026-06-06
- 平台：带屏 AI 眼镜 · AIUI Runtime
- 角色分工：PM 负责需求 / 流程 / 两个确定性模块 / 工程规格；**分镜模板是死模板（静态资产），生产只调用；现阶段经一次性引导环节先把它生成出来**

---

## 1. 产品定义

用户用一句话语音，说出**想拍的画面**与**想表达的情绪**；系统解析意图后，分两条独立管线分别处理「画面 → 构图」与「情绪 → 剪辑节奏」，再把两者组合成一套 3–6 镜的短样片分镜方案，并为每个分镜生成一张**两层色（黑底 + 荧光绿）的剪辑分镜示意图**，最后给出逐镜可执行的分镜指引。

**输入输出基线（已锁定）**

- 输入：全部收敛为**语音**。
- 输出：官方 UI 范式，以 **Markdown 卡片 + 两层色 Canvas 构图图**呈现。
- 调度：调用官方文档定义的能力（见第 9 节）。

---

## 2. 端到端流程（意图在前，再并行）

> 重要：意图 / 情绪识别与分镜分析**不能并发**。意图分析是顺序前置阶段，**追问发生在这一步**；
> 拿到足够 context 后，才拆成两条并行管线。

```
①用户描述：说出想要的画面 + 想表达的情绪
        │
        ▼
②分析用户意图  ← 追问发生在这里
   抽取：主体 / 场景 / 画面意图 / 情绪基调
   信息不足 → 智能追问（1~2 轮）直到 context 足够
        │  （顺序，不并行）
        ▼  context 足够后才拆分
   ┌────┴────────────────┐   ← 此处才并行，两个独立 LLM
   ▼                     ▼
管线(a) 画面 → 构图        管线(b) 情绪 → 剪辑节奏
  组关键词→调用死模板        读【模块B 剪辑节奏】映射节奏
   └────┬────────────────┘
        ▼
③组合输出：每个构图 + 该构图下画面的切换速度
        ▼
④两层色剪辑分镜示意图：构图（空间）＋ 节奏（时间）综合成一张图
        ▼
⑤分镜指引：逐镜可执行的拍摄指引
```

> 管线 a 的本质：环节 ② 之后，LLM **分析 → 组成关键词 → 用关键词调用死模板**来输出构图；
> 生产运行时不生成、不改写模板（模板是固定静态资产，见第 4 节）。

---

## 3. 已锁定的产品决策

| 维度 | 定稿 | 说明 |
|---|---|---|
| 线A 追问 | **智能判断** | 信息足不问；缺关键项（主体/场景/情绪三选一缺失）才反问，上限 1~2 轮 |
| 模板使用 | **跨模板混搭** | 不锁单一模板，按镜从模板库抽取组合；由情绪统一调和 |
| 分镜规模 | **短样片 3~6 镜** | 组合层硬约束 `3 ≤ 镜数 ≤ 6` |
| 构图图配色 | **AIUI 绿** | 两层色固定：底 `#000` + 形 `#40FF5E`（Canvas 内写死 hex，不用 token） |

**跨模板混搭护栏**：一次方案里运镜手法收敛在 **2~3 种**以内、情绪基调单一——模板可混，但节奏与色调由线A统一锁死，保持「统一视觉语言」。

## 4. 分镜模板规格（死模板 / 静态资产）

> **核心原则**：分镜模板是**死模板**——一份固定的静态资产。生产运行时**不生成、不改写**模板，
> 只用关键词**调用**它来输出。
>
> 之所以 PRD 里仍保留「模板生成」环节（见 8.1 A0），**只是因为现在还没有这套分镜模板**：需要一个
> **一次性引导（bootstrapping）环节**，把死模板先生成出来、落盘成固定资产。生成完成后该环节即退场，
> 生产流程只读取、调用，绝不在运行时触碰模板生成。

### 4.1 模板 Schema（每个模板必含 4 字段）

| 字段 | 名称 | 内容要求 |
|---|---|---|
| 1 | **图片构图** | 该构图本身必须是一个**二维图形 / 形状**（两层色：底 #000 + 形 #40FF5E）。即可直接绘制的构图示意，而非文字。来源对照【模块A 分镜构图】的 10 种构图画法。 |
| 2 | **分镜说明** | 简要说明该分镜构图：构图法则（三分法 / 二分法 / 中心…）+ 景别（近景 / 远景…）+ 镜头调度（固定 / 摇臂 / 移动…）。 |
| 3 | **决策指引** | 给 LLM 参考的选用说明：什么时候该调用这个分镜、它解决什么画面问题。供管线 a 的构图 Agent 决策。 |
| 4 | **场景适配** | 说明该镜头适配的**情绪**与**场景**（如 情绪=温暖；场景=居家口播 / Vlog 自述）。 |

### 4.2 模板示例（编号格式，Agent 产出参照此样式）

```
模板 #01
1. 图片构图：[二维图形] 三分线网格 + 左下交点处实心主体块（黑底荧光绿）
2. 分镜说明：三分法 · 中景 · 固定
3. 决策指引：人物口播 / 对话、需要稳定与呼吸感时选用；主体偏置、留出视线空间
4. 场景适配：情绪=温暖 / 真诚；场景=居家口播、采访、Vlog 自述

模板 #02
1. 图片构图：[二维图形] 大面积空底 + 右下角一个极小主体点
2. 分镜说明：极远景渺小主体 · 留白 · 固定 / 航拍
3. 决策指引：表达孤独、渺小、史诗感，或做段落留白呼吸时选用
4. 场景适配：情绪=忧郁 / 史诗壮阔；场景=风光、山野、桥/崖边

模板 #03
1. 图片构图：[二维图形] 两条向中心汇聚的斜线 + 远端小主体（灭点）
2. 分镜说明：引导线 / 灭点 · 中景→远景 · POV 主观
3. 决策指引：表达出发、纵深、代入感；公路 / 通道类画面首选
4. 场景适配：情绪=自由 / 期待；场景=公路、走廊、行进中
```

### 4.3 模板库与覆盖

- 一次性引导环节以【模块A】的 10 种构图为基础，叉乘【受控词表】的景别 / 镜头调度 / 情绪场景，生成一组**编号死模板**并落盘。
- 生成后即为**固定静态资产**；生产运行时只按关键词调用，不再改写。若需扩充，重跑引导环节即可——但这属于**离线维护**，不在生产流程内。
- 第 5 节的 8 个分镜原型（T1–T8）作为引导环节的**风格参考**，不直接进生产。

## 6. 两个确定性模块（PM 负责）

| 模块 | 职责 | 文档 |
|---|---|---|
| 模块A 分镜构图 | 把素材压缩成 10 种规范构图，各配两层色示意定义；供模板「图片构图」字段引用 | [模块A-分镜构图.md](./模块A-分镜构图.md) |
| 模块B 剪辑节奏 | 整理「情绪 → 剪辑节奏」映射；供管线 b 直接调用 | [模块B-剪辑节奏.md](./模块B-剪辑节奏.md) |

**组合规则 → 剪辑分镜示意图**：同一镜叠加两份数据 —

```
某一镜 = 模板库给的【构图二维图形】 + 模块B 给的【时长 / 切换速度 / 转场】
```

落到两层色示意图（仅 #000 + #40FF5E）：
- **图底层**：画构图（模板的图片构图字段）——“怎么摆”。
- **节奏层**（同色，叠加框外）：节奏条长度 ∝ 时长；镜间转场符号（硬切=竖线 / 叠化=交叉 / 甩切=斜纹）；切换速度由条的密度体现。

## 7. 最终输出结构

每次推荐返回：标题 + 情绪基调 + 所用模板编号（混搭列表）+ 分镜数组。分镜数组每一项：

- **(a) 分镜数量** = 数组长度，卡片顶部明示（3~6）。
- **(b) 每镜时长**（秒）。
- **(c) 串联剪辑** = 独立数组，逐镜列「本镜 →(转场)→ 下一镜」。
- **(d) 简要说明** = 一句话画面 + **镜头调度语言** + 景别 + 构图。
- 外加：每镜的**两层色剪辑分镜示意图**。

## 8. 工程开发（重点）

整套系统按「Agent 层 + UI 层」两层开发。Agent 层产出数据，UI 层消费数据。

### 8.1 Agent 层

共 1 个离线 Agent + 4 个生产 Agent。生产 Agent 间的顺序 / 并行边界严格按第 2 节。

| 代号 | Agent | 时机 | 输入 | 处理 | 输出 | 依赖 |
|---|---|---|---|---|---|---|
| **A0** | 模板生成（一次性引导，**非运行时**） | 仅项目启动、模板尚不存在时执行一次 | 模块A + 词表 + 8 原型 | 按 4.1 schema 生成编号**死模板**并落盘 | **死模板库（固定静态资产）** | 模块A、第 5 节 |
| **A1** | 意图分析 | 生产·顺序前置 | 语音文本 | 抽取 主体/场景/画面意图/情绪；缺关键项追问（≤2 轮） | 结构化意图 + 情绪基调 | — |
| **A2** | 画面构图（管线 a） | 生产·并行 | 结构化意图 | LLM 分析 → 组成关键词 → 用关键词调用死模板（按决策指引/场景适配选用），跨模板混搭 | 每镜构图（引用死模板编号） | 死模板库 |
| **A3** | 剪辑节奏（管线 b） | 生产·并行 | 情绪基调 | 模块B 映射 | 每镜 时长/切换速度/转场 | 模块B |
| **A4** | 组合 | 生产·汇合 | A2 + A3 | 按镜对齐、约束 3~6 镜、生成示意图数据 + 分镜指引 | 最终分镜数组 + 示意图数据 | — |

要点：
- **A0 是一次性引导环节**：只在死模板尚不存在时跑一次，落盘后退场；生产流程**不含**任何模板生成。
- **A1 必须先完成且 context 足够**，A2 / A3 才启动（不可并发于 A1）。
- A2 与 A3 **并行**（AIUI 端侧 LanguageModel 用两个独立会话）。
- A2 **只按关键词调用死模板、绝不现编或改写构图**；构图二维图形来自固定的死模板库。
- A4 是唯一产出对外数据结构（第 7 节）的地方。
- 输出走稳健 JSON 解析（端侧 LanguageModel 的结构化 tool-call 当前未在 JS 层暴露）；受控词表注入各 Agent 的 system prompt 约束取值。

### 8.2 UI 层

单页应用，按状态机驱动；组件全部用官方范式。

| 状态 | 画面 | 组件（AIUI） |
|---|---|---|
| 待命 | 麦克风按钮 + 提示语 | `button` |
| 聆听 | 语音波形 / 实时转写 | `SpeechRecognition` + `text` |
| 追问 | 展示 A1 的追问，重新聆听 | `streamdown` + `button` |
| 分析中 | 进度：意图 → 并行(构图/节奏) → 组合 | `text` / 状态点 |
| 结果 | Markdown 卡片（标题/情绪/模板编号/分镜数组 a–d）+ 逐镜两层色示意图 + 分镜指引列表 | `streamdown` + `canvas` + `scroll-view` |
| 错误 | 错误提示 + 重试 | `error-state` + `button` |

要点：
- 两层色示意图用 `<canvas>` + `wx.createCanvasContext`，颜色写死 `#000` / `#40FF5E`（Canvas 不支持 token）。
- 多镜示意图建议绘制为一条**胶片条**（每格一镜：构图 + 节奏条 + 镜间转场符号），放进 `scroll-view`。
- 设计规范：480px 宽、黑底、卡片式、12px 圆角、2px 边框、无 emoji（见第 9 节）。
- 结果文字部分由 A4 数据**确定性生成 Markdown**（不依赖流式解析），保证稳定渲染。

## 9. 落地平台对照（AIUI 能力映射）

| 流程环节 | AIUI 官方能力 |

| 两层色示意图 | `<canvas>` + `wx.createCanvasContext`（hex：#000 / #40FF5E） |

## 10. 受控词表

- **景别**：远景 / 全景 / 中景 / 近景 / 特写 / 大特写
- **镜头调度语言**：固定 · 摇镜 · 俯仰 · 移镜(推/拉) · 跟拍 · 摇臂 · 手持 · 变焦 · 航拍 · POV主观
- **构图法则**：中心 · 三分 · 二分 · 对称/地平线 · 极远景渺小主体 · 前景框架 · 引导线/灭点 · 低角度仰拍 · 俯拍顶视 · 侧面逆光剪影 · 浅景深特写
- **转场/剪辑**：硬切 · 叠化 · 淡入淡出 · 匹配剪辑 · 甩切 · 卡点快切 · J切/L切
- **情绪基调**：平静治愈 / 温暖 / 忧郁 / 紧张 / 史诗壮阔 / 欢快 / 高级质感 / 动感
