## 项目一句话

带屏 AI 眼镜上的语音分镜助手：用户语音描述「想拍的画面 + 想表达的情绪」→ 解析意图（可追问）→ 并行分析（画面→构图 / 情绪→剪辑节奏）→ 组合 → 输出 3~6 镜分镜方案 + 每镜的两层色剪辑分镜示意图 + 分镜指引。

## 必须吃透的硬约束（来自 PRD，不可改动）

1. **流程顺序**：意图分析在前（顺序、含追问），context 足够后才拆**两条并行管线**；不可把意图与分镜并发。
2. **死模板**：分镜模板是**固定静态资产**，生产运行时**只按关键词调用、不生成不改写**。`A0` 是**一次性引导环节**，仅因现在还没有这套模板才需要——造完落盘退场，生产流程不含模板生成。
3. **模板 schema**（每条死模板 4 字段）：①图片构图（二维图形/形状本身，两层色）②分镜说明 ③决策指引（给 LLM 选用）④场景适配（情绪+场景）。
4. **两层色**：示意图仅 `#000`（底）+ `#40FF5E`（形）两色；Canvas 内颜色写死 hex，不能用 token。
5. **分镜规模**：硬约束 3~6 镜。
6. **跨模板混搭**护栏：单次方案运镜手法 ≤2~3 种、情绪基调单一。
7. **全部能力走 AIUI 官方 API**：语音 `SpeechRecognition`；各 Agent 端侧 `LanguageModel`（A2/A3 用两个独立会话并行）；文字输出 `<streamdown>`；示意图 `<canvas>`+`wx.createCanvasContext`；设计规范 480px/黑底/卡片/12px 圆角/2px 边框/无 emoji。

## Agent 层（PRD §8.1，需在架构里细化）

- A0 模板生成（一次性引导，非运行时）→ 落盘**死模板库**
- A1 意图分析（顺序前置，含 ≤2 轮追问）
- A2 画面构图（并行管线 a）：LLM 分析→组关键词→**调用死模板**
- A3 剪辑节奏（并行管线 b）：调用【模块B】映射
- A4 组合：对齐两路数据、约束 3~6 镜、产出最终数据结构 + 示意图数据 + 分镜指引
